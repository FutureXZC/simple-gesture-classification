31 /*
32  * \Desc Plugin implementation
33  * Message processing includes the upper part and the lower part:
34  * 1) The upper part receives bus messages and pushes them to the local queue;
35  * 2) The local execution engine schedules the lower part to complete the
36  * distribution and processing of messages
37  *
38  * @brief Two basic classes XPlugin and XPluginAsync are implemented. To
39  * implement a custom plugin, you only need to inherit any one of them. It is
40  * recommended to choose the XPluginAsync class. There are some differences
41  * between the two basic classes, which are explained as follows: XPluginAsync
42  * is a subclass of XPlugin. XPluginAsync implements the asynchronous processing
43  * mechanism of messages. XPlugin provides interfaces for message distribution,
44  * get & set the upper limit of message number, get & set message processing
45  * timeout time, etc. The subclasses implement the corresponding interface to
46  * implement the plugin function. XPluginAsync inherits from XPlugin and
47  * implements functions such as message distribution, flow control, and message
48  * processing timeout monitoring. Subclasses only need to pay attention to the
49  * plugin's own functions.
50  */
