: xproto message type registration interface.In addition to implementing a unified interface, xproto messages also need to register their message types with xproto. Each type of message has a message type and structure in the form of a string. This interface is a macro. The parameter "key" is used to indicate the declared message type. It needs to be written directly in the format of the identifier, and the macro will convert it into a string internally. Note: You need to call this interface to declare the message type before the consumer Plugin calls the subscription message interface. Generally, this macro is placed in the position of the global variable declaration.
